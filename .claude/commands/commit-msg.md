# [コミット] 日本語コミットメッセージを生成（ステージ差分）

## コマンド: /commit-msg [$ARGUMENTS]
ステージ済み差分（`git diff --staged`）を根拠に、**日本語のコミットメッセージ**を生成する。  
Cursor/GitLens等の自動提案が英語になる場合の代替として使う。

## 入力: $ARGUMENTS（任意）
- 任意: 追加で伝えたい意図（例: `なぜこの修正が必要か` / `ユーザー影響`）

---

## 🎯 目的
- コミットメッセージを **原則日本語**に揃える（`AGENTS.md` の規約に従う）
- 変更内容を短く正確に要約し、必要なら背景/注意点を本文で補足する

---

## 前提（必須）
- ステージされている変更があること（`git add` 済み）
  - ステージが空なら、まずユーザーに「ステージしてから再実行」を案内してここで停止

---

## 進め方（固定）
1. `git status` と `git diff --staged` を確認する（ステージ差分のみを根拠にする）
2. 変更の目的を1つに要約する（混ざっていたら「コミットを分ける」を提案する）
3. **コミットメッセージを日本語で生成**する

---

## 出力フォーマット（必ずこの形）
- 1行目（50〜72文字目安）: `<type>: <日本語の要約>`
  - type例: `fix` / `refactor` / `docs` / `feat` / `chore`
- 本文（必要なときだけ、箇条書き）:
  - 背景/意図/注意点（レビュー観点になる情報）

出力は **コミットメッセージ本文だけ**（余計な解説やコードは出さない）。
