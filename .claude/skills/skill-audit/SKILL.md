# [スキル管理] スキルオーディット

## 入力: $ARGUMENTS
- 省略可（全スキルを対象）
- カテゴリ指定: `design`, `task`, `bug`, `review` 等

---

## 🎯 目的
- **スキル一覧を分析**して、重複・類似・改善候補を特定する
- 使用頻度の低いスキルの統廃合を提案する
- スキル構成の健全性を可視化する

---

## 自動化できる範囲

| 項目 | 自動化 | 備考 |
|------|--------|------|
| スキル一覧取得 | ✅ | `.claude/skills/*/SKILL.md` |
| 目的・説明の抽出 | ✅ | descriptionと🎯目的セクション |
| 重複・類似検出 | ✅ | 目的の類似度を比較 |
| 更新日時確認 | ✅ | gitログから取得 |
| 使用頻度 | ❌ | ユーザー申告が必要 |
| 統廃合判断 | ❌ | ユーザー承認が必要 |

---

## 実行手順

### 1. スキル一覧取得

```bash
# スキルディレクトリ一覧
ls -1 .claude/skills/

# 各スキルのdescription取得
for dir in .claude/skills/*/; do
  name=$(basename "$dir")
  desc=$(grep -A1 "^description:" "$dir/SKILL.md" | tail -1 | sed 's/^description: "//' | sed 's/"$//')
  echo "$name: $desc"
done
```

### 4. 更新日時チェック

```bash
# 各スキルの最終更新日
for dir in .claude/skills/*/; do
  name=$(basename "$dir")
  date=$(git log -1 --format="%ai" -- "$dir" 2>/dev/null || echo "untracked")
  echo "$name: $date"
done
```

---

## 注意事項

- **削除・統合は必ずユーザー承認後**に実行する
- 使用頻度はユーザーの記憶に依存するため、不確実な情報として扱う
- スキルの目的が明確に異なる場合は、名前が似ていても統合しない
