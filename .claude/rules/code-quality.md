# AI運用ガイドライン（CLAUDE.md補足）

## サンプルコード運用
- 原因切り分け・最小検証のためだけに作成（プロダクション流用しない）
- 配置: 実行系から参照可能な `samples/` に切る。本番ビルド・配布からは除外
- 命名: `sample_{short-purpose}.{ext}`、先頭コメントに「一時利用・削除可」明記
- 実鍵・個人情報・API Key・本番エンドポイント禁止（モック/ダミー値）
- ビルド/テスト/カバレッジから除外。解決後は同PRで削除

## 依存（ライブラリ）評価の補助基準
- 評価観点: 直近リリース日、Issue応答、ドキュメント質、DL数/週、互換性、代替案の有無

## ADR-lite（変更要求）テンプレ
- Context: 何が前提で何が問題か
- Decision: 何を決めるか（選択肢A/Bと比較）
- Consequences: 影響（メリット/デメリット/運用/コスト）
- Alternatives: 採用しない案と理由
