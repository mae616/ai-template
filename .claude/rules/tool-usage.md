# ツールエラー防止

- **ファイル操作前に存在確認**: `Edit`の前に`Read`で対象ファイルの存在と内容を確認する
- **長時間コマンドのtimeout**: 10秒以上かかりそうなBashコマンドは`timeout`パラメータを明示する
- **破壊的操作の事前確認**: `git push --force`, `git reset --hard`, `rm -rf`等は実行前にユーザーへ意図確認
- **パス存在チェック**: ディレクトリ作成・ファイル書き込み前に親ディレクトリの存在を確認
- **Pencil（.pen）ファイルの保存指示**: Claude CodeはPencilの保存操作を実行できない。デザイン作業の区切り（各ステップ完了時）で、ユーザーに**保存先パス**（原則 `doc/input/design/ファイル名.pen`）と**保存操作の実行**を明示的に指示する。保存確認後に次のステップへ進む。
- **対話式コマンドのガイド**: `npm create`、`npx create-*-app` 等の対話式プロンプトを含むコマンドはユーザーが手動実行する。AIは事前に以下を提示する:
  1. 実行するコマンド全文
  2. 各プロンプトで**推奨する選択/入力値**とその理由
  3. 「全選択」等のまとめ選択がある場合、**選択すべき項目・外すべき項目**を明示
