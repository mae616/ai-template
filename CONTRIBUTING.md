## 🚀 開発環境のセットアップ

### 前提条件
- Git
- Podman 5.5.2以上
- podman-compose 1.5.0以上
- VS Code / Cursor

### セットアップ手順

1. **リポジトリをフォーク**
   ```bash
   # GitHubでリポジトリをフォーク
   # その後、ローカルにクローン
   git clone https://github.com/your-username/ai-template.git
   cd ai-template
   ```

2. **リモートの設定**
   ```bash
   # 上流リポジトリを追加
   git remote add upstream https://github.com/mae616/ai-template.git
   
   # リモートの確認
   git remote -v
   ```

3. **DevContainerで開発環境を起動**
   - VS Code/Cursorでプロジェクトフォルダを開く
   - 「Reopen in Container」を選択
   - 環境のセットアップ完了を待つ

4. **依存関係のインストール**
   ```bash
   # ツールの確認
   mise --version
   node --version
   python --version
   
   # 必要に応じてツールをインストール
   mise install
   ```

## 🔄 プルリクエストの手順

### 1. ブランチの作成
```bash
# 最新のmainブランチを取得
git fetch upstream
git checkout upstream/main

# 新しいブランチを作成
git checkout -b feature/your-feature-name
```

### 2. 変更の実装
- コードの実装
- テストの追加
- ドキュメントの更新
- コーディング規約の遵守

### 3. コミット
```bash
# 変更をステージング
git add .

# コミット
git commit -m "feat: 新機能の追加

- 機能の詳細説明
- 変更理由
- 影響範囲"
```

### 4. プッシュ
```bash
# フォークしたリポジトリにプッシュ
git push origin feature/your-feature-name
```

### 5. プルリクエストの作成
- GitHubでプルリクエストを作成
- タイトルは簡潔に、説明は詳細に
- 関連するIssueをリンク
- レビュアーを指定

### コミットメッセージの規約

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type
- **feat**: 新機能
- **fix**: バグ修正
- **docs**: ドキュメント
- **style**: コードスタイル
- **refactor**: リファクタリング
- **test**: テスト
- **chore**: その他の変更

#### Scope
- 影響を受ける範囲（例: api, ui, auth）

#### Subject
- 50文字以内で簡潔に

#### Body
- 変更の詳細説明
- 変更理由
- 影響範囲

#### Footer
- 関連するIssue番号
- 破壊的変更の説明
